# ADR-006: Security Baseline and Secret Boundaries

- Status: Accepted
- Date: 2026-03-05
- Owners: Security Team
- Related: docs/architecture/public-baseline.md Sections 2 and 5, AGENTS.md Section 10

## Context
The system processes sensitive enterprise evidence and governance documents across tenants.

## Decision
Security baseline:
- Managed identity for service-to-service auth where possible
- Azure Key Vault for secrets
- Network and endpoint allowlists for model and data services
- Tenant isolation checks on every scoped query

## Consequences
- Stronger least-privilege posture
- Clear incident and audit boundaries

## Verification
- Security tests must validate tenant-isolation failure cases.
- CI security checks must run on app and infra definitions.
