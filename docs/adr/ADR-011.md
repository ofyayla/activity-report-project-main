# ADR-011: Dependency and Package Governance

- Status: Accepted
- Date: 2026-03-05
- Owners: Engineering Productivity, Security
- Related: docs/architecture/public-baseline.md Section 6

## Context
Multi-service stack requires controlled dependency lifecycle to avoid drift and security regressions.

## Decision
Governance policy:
- Required packages and stacks are locked by PLAN Section 66
- Major versions are pinned before implementation phase
- Package additions/removals require ADR update and architecture approval

CI gates:
- fail on critical CVEs
- fail on license policy violations
- fail on contract/schema breakages

## Consequences
- Better supply-chain control
- Slightly slower dependency upgrade velocity

## Verification
- CI checks include dependency, license, and contract validation stages.
- Release checklist includes monthly dependency review evidence.
