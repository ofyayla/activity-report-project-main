# ADR-012: Premium Dashboard UI and Motion Guardrails

- Status: Accepted
- Date: 2026-03-05
- Owners: Product Design, Frontend Team
- Related: docs/architecture/public-baseline.md Section 7

## Context
The board dashboard must be visually premium, trustworthy, fast, and auditable.

## Decision
UI guardrails:
- Typography: Geist Sans + Geist Mono
- Token-based theming with tenant overrides
- Motion is meaningful only (state/progress communication)
- Reduced-motion support is mandatory
- Dashboard narratives can only use PASS-verified claims

Performance and accessibility:
- Web vitals targets: LCP <= 2.5s, INP <= 200ms, CLS <= 0.1
- Accessibility target: WCAG 2.2 AA

## Consequences
- Strong UX consistency and trust
- Requires strict component and motion review discipline

## Verification
- Dashboard QA must include reduced-motion and keyboard navigation checks.
- Performance budget checks must run in staging.
