# ADR-010: Observability SLOs and Audit Telemetry

- Status: Accepted
- Date: 2026-03-05
- Owners: SRE, Platform Team
- Related: docs/architecture/public-baseline.md Section 8, AGENTS.md Section 11

## Context
End-to-end traceability and reliability targets require unified telemetry and SLO ownership.

## Decision
Observability baseline:
- OpenTelemetry instrumentation
- Azure Monitor / Application Insights sink
- Structured logs with tenant_id, project_id, run_id

SLO domains:
- report-run success rate
- node latency p95
- approval SLA breach rate
- queue lag and retry depth

## Consequences
- Faster incident diagnosis
- Additional telemetry overhead

## Verification
- Traces and logs must be correlated by run_id.
- SLO dashboards must show p50/p95 and error budgets.
