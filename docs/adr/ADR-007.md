# ADR-007: Verifier Gate Thresholds and Publish Blocking

- Status: Accepted
- Date: 2026-03-05
- Owners: AI Quality Team, Compliance
- Related: docs/architecture/public-baseline.md Sections 3 and 8, AGENTS.md Section 9

## Context
Zero-hallucination objectives require strict claim-level verification and deterministic publish control.

## Decision
Verifier status model:
- PASS
- FAIL
- UNSURE

Hard publish blockers:
- Critical FAIL claim
- Missing citation on any claim
- Missing calculator artifact on numeric claim
- Policy violations or required approvals missing

## Consequences
- High trust in published outputs
- More review loops in ambiguous cases

## Verification
- Integration tests must prove publish blocked on each hard blocker.
- Verifier outputs must include reason and source references.
