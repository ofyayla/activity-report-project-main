# ADR-002: Retrieval Engine Baseline and Fallback

- Status: Accepted
- Date: 2026-03-05
- Owners: Platform Team, AI Team
- Related: docs/architecture/public-baseline.md Sections 3 and 5, AGENTS.md Section 3.4

## Context
The system needs hybrid retrieval with reliable metadata and enterprise operations.

## Decision
Default production retrieval engine:
- Azure AI Search (hybrid: dense + sparse)

Local non-production fallback:
- PostgreSQL + pgvector only for controlled development scenarios

## Consequences
- Production consistency and managed scaling
- Clear non-production fallback policy

## Verification
- Retrieval service config must default to Azure AI Search.
- Fallback mode must be environment-gated and disabled in production.
