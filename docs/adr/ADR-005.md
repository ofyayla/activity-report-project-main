# ADR-005: Data Architecture Split (OLTP + Analytics Read Model)

- Status: Accepted
- Date: 2026-03-05
- Owners: Backend Team, Data Team
- Related: docs/architecture/public-baseline.md Sections 4 and 5

## Context
Operational workflows and dashboard analytics have different latency and query shape requirements.

## Decision
Data architecture split:
- OLTP model for transactional workflows (runs, approvals, claims, verification)
- Analytics read model for dashboard-heavy aggregate queries

## Consequences
- Better performance isolation
- Clear ownership of operational consistency vs analytical speed

## Verification
- API contracts separate transactional endpoints from aggregate KPI endpoints.
- Dashboard latency SLO checks use read-model endpoints.
