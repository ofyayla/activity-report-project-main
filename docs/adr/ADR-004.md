# ADR-004: Dashboard and Chart Stack Standard

- Status: Accepted
- Date: 2026-03-05
- Owners: Product Design, Frontend Team
- Related: docs/architecture/public-baseline.md Sections 5 and 7

## Context
Board-grade ESG cockpit requires high-density visuals, traceability drilldowns, and consistent design tokens.

## Decision
Production UI/visualization stack:
- Next.js + TypeScript + Tailwind + shadcn/ui
- ECharts + echarts-for-react as the single production chart engine
- Motion runtime: motion/react

Prototype-only note:
- Recharts examples may be used only outside production paths.

## Consequences
- Consistent visual system
- Lower maintenance overhead from single chart stack

## Verification
- Dashboard components must use shared token adapters.
- Chart implementations in production routes must use ECharts wrappers.
